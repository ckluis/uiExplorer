Agent Columns

v3

Purpose

Adds model selection visibility, per-agent token usage, and cost tracking to the orchestration dashboard. Each agent shows which model powers it (Opus, Sonnet, Haiku), how many tokens it consumed, and the cost — with rollups at the manager and orchestrator level. All math uses realistic Anthropic API pricing.

Prompt

One of the difficulties in orchestration is using the right model for each task/subagent. I think its worth showing which agent is being used at each step, how many tokens used, and cost.

And it should roll-up to each manager and the overall orchestrator.

Make sure the math makes sense.
AskUserQuestion
Should the cost math use realistic Anthropic API pricing (Opus $15/$75, Sonnet $3/$15, Haiku $0.25/$1.25 per 1M tokens), or do you want simplified/round numbers that are easier to scan?
Realistic Anthropic pricing (Recommended) — Uses actual per-model rates, costs will have cents precision like $0.87
Simplified round numbers — Round to nearest nickel/dime for cleaner visual, less precise
Answer: Realistic Anthropic pricing (Recommended)
