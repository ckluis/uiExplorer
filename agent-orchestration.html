Purpose

This is the origination prompt for a 20-variant research series (two waves) exploring radically different UI paradigms for human/agent interaction — making orchestration, sub-agents, and recursive agent reasoning visible and beautiful.

Origination Prompt #1

This page documents the master prompts and research context that generated 20 autonomous human/agent orchestration variants across two waves. The first wave (v1-v10) explored fundamentally different visual metaphors. The second wave (v11-v20) enhanced each with device preview frameworks and CSS-only interactions.

Verbatim Origination Prompt
Let's evaluate 10 experimental designs for human/agent interaction. We see chat, cli, email, slack, kanban as interfaces, but it feels like NONE of them do a good job of showing off orchestration, sub-agents, sub-sub-agents, etc.

The important thing here is to GO WILD. Do deep research first, but not let it confine your thinking. Each approach should be WILDLY different from the current approaches. We want to wow people with unique takes on it. The prompts should be extra specially clear on why each variant is important.

Title this experiement: Human/Agent Orchestration.

We need to consider tools like "AskQuestionUserTool" that would be part of these interactions and the value of agents kicking off subagents just to ask better questions with potential answers. Agents are soo much better at thinking.

Go NUTS!

Research Domains

Agent Orchestration Patterns

Multi-agent systems, orchestrator/worker patterns, recursive delegation, fan-out/fan-in, agent-to-agent communication protocols.

Human-AI Interaction Research

Conversational UI patterns, progressive disclosure, mixed-initiative interaction, trust calibration, mental models of AI systems.

Nature's Orchestration Systems

Mycelial networks, neural pathways, ocean ecosystems, particle physics, kitchen brigades — natural metaphors for coordination and delegation.

Information Visualization

Edward Tufte's principles, network graphs, temporal visualizations, layered information architecture, sparklines and small multiples.

Sci-Fi & Speculative Interfaces

Mission control rooms, orchestra conductors, cartographic expeditions, weaving looms — metaphors that make invisible processes visible.

Cognitive Psychology

How humans understand hierarchical processes, attention management, information foraging theory, progressive revelation of complexity.

Execution Model

1 Research Agent — Deep research across agent orchestration, HCI, nature's coordination systems, and information visualization
↓

↓
3 Validation Agent — Verifies all links, patterns, and consistency across the series

Design Constraints (Wave 1)

Every variant follows the UI Explorations framework rules: pure HTML/CSS, no JavaScript, no external dependencies beyond Google Fonts. Each includes the standard experiment sidebar and depicts a frozen moment of an agent orchestration in progress — showing multiple levels of agent hierarchy, data flow, and the AskUserQuestion pattern rendered through each metaphor's unique visual language.

Origination Prompt #2

After completing the first 10 variants, a second origination prompt was issued to create enhanced iterations (v11-v20). Each takes its corresponding v1-v10 variant and adds the device preview window framework and semi-functional CSS-only interactions.

Verbatim Origination Prompt #2
We need to add new versions of all 10 agent orchestration (which will be v11-v20), but will require a second origination prompt. This leads me to believe we will need to add Origination prompts inline to the cards so they can occur between autonomous generation.

The new origination prompt for these new versions:

1. each one needs to exist within the "window" framework and have desktop, mobile, full screen
2. each version needs to be iterated to have a semi-functional ui with a couple of click/hover interactions

This can be broken into a meta changelog for improvements to experiments.html and then also parsed into the 10 specific prompts that will be used to modify each original design according to the origination prompt instructions. Remember to carefully log the new generated prompt for each version (which should mention the original file it is modifying).

Remember documentation is key.

Design Constraints (Wave 2)

Same base constraints as Wave 1, plus: each variant must include the device preview switcher (desktop 1360×820, mobile 375×720, fullscreen), CSS-only interactions using checkbox hack, :hover, :focus-within, and radio buttons. Mobile layouts must adapt meaningfully, not just shrink.
