Purpose

Support Toolkit variant exploring a split-pane layout for support agents and CSMs helping customers. Shows customer context alongside issue checklist with resolution scripts. Part of a 10-variant research series.

Variant Focus

The "helping someone else" mental model. Unlike other variants where users fix their own issues, this variant is designed for support agents who need customer history, account context, and resolution scripts to help customers resolve their problems.

Prompt

This version was generated autonomously by a sub-agent. The prompt below is the exact prompt given to the build agent — it was crafted by the orchestrating agent based on the origination prompt and research phase.

Build guided-checklists-v7.html — the "Support Toolkit" variant for the Guided Checklists experiment series.

Name: Support Toolkit
Tagline: "Here's what's wrong with their account."
Version: v7

Core Concept: A support agent's split-pane view showing customer context alongside issue checklist. Designed for CSMs and support agents helping customers. Shows history, related tickets, and resolution scripts.

Exposure Method: Dedicated toolbar button when in "Support Mode". Always visible when viewing a customer account.

Presentation Style: Split-pane layout. Left: customer context (profile, history, tickets). Right: issue checklist with resolution scripts.

Target Persona: Support agent, customer success manager. Helping someone ELSE fix their issues. Needs customer context and history.

Platform: Web (optimized for desktop side-by-side with ticketing system).

Key UI Elements:
- Split pane (40/60 ratio)
- Customer profile card (name, company, plan tier badge)
- Customer timeline (recent activity)
- Related ticket links
- Issue checklist with resolution scripts
- "Copy to ticket" button on each issue
- Internal notes (support-only, not visible to customer)
- Plan tier indicators (affects available fixes)
- "Last contacted" and "Account health" badges

Sample Customer Context:
Customer: Sarah Johnson, Acme Corp
Plan: Enterprise (green badge)
Health Score: 72/100
Last Contact: 3 days ago (phone call)
Open Tickets: 2
Recent Activity: Changed password, Updated billing, API error logged

Sample Issues for This Customer:
1. "API rate limit hit 3 times this week" — Script: "Check their integration setup, may need plan upgrade"
2. "SSO configuration incomplete" — Script: "Walk through SSO wizard, common issue with Okta"
3. "Team member invite pending 7 days" — Script: "Resend invite or check spam folder"

Color/Mood: Professional, efficient. Support tool aesthetic. Green for customer health, amber for issues.

Interaction Patterns:
- Radio buttons for customer/ticket selection
- Checkbox for expand/collapse resolution scripts
- Focus-within for "Copy to ticket" highlight
- Hover on timeline for details

Device Preview: Include desktop/mobile/fullscreen switcher.
