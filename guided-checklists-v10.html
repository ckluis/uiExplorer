Purpose

Explores an "Ambient Intelligence" approach where the system automatically fixes issues and only notifies users AFTER the fact. Undo-centric rather than approval-centric. Part of a 10-variant research series.

Build guided-checklists-v10.html — the "Ambient Intelligence" variant for the Guided Checklists experiment series.

Name: Ambient Intelligence
Tagline: "We fixed it. Undo if we got it wrong."
Version: v10

Core Concept: The system automatically fixes issues and only notifies users AFTER the fact. Undo-centric rather than approval-centric. For users who hate interruptions — they see toasts telling them what was auto-fixed with a countdown to undo.

Exposure Method: Thin status bar indicator at top showing system status. Toast notifications appear when auto-fixes happen.

Presentation Style: Expandable snackbar/toast. Compact by default, expands to show details. Undo button with countdown.

Target Persona: Regular user who hates interruptions. Trusts the system, just wants to be informed (not asked). Prefers undo to approval.

Platform: Both web and mobile.

Key UI Elements:
- Thin status bar ("All systems normal" / "2 issues auto-fixed")
- Toast notifications sliding up from bottom
- Expand arrow on toasts to see details
- 10-second undo countdown (visual progress bar)
- "Don't auto-fix this type" learning button
- Stacked toasts for multiple auto-fixes
- Green checkmark for confirmed fixes
- "Learn more" link in expanded state

Sample Auto-Fix Toasts:
1. "Refreshed expired API token" — [Undo 8s] [Details]
2. "Cleared stale cache (saved 2GB)" — [Undo 5s]
3. "Reconnected Slack webhook" — [Undo 10s]
4. "Updated timezone to match browser" — [Undo 3s]

Color/Mood: Calm, minimal, confident. Green for success. Unobtrusive but informative. Trust-building aesthetic.

AI Feedback Pattern:
- "Undo" teaches what NOT to auto-fix
- Letting countdown expire = confirming good fix
- "Don't auto-fix this type" = explicit training
- "Learn more" = wanting more control

Interaction Patterns:
- Checkbox for toast expand/collapse
- CSS animation for countdown progress bar (depleting)
- Toast stacking with transform translateY
- Status bar :hover shows details
- Radio buttons for undo state (pending/undone/confirmed)

Device Preview: Include desktop/mobile/fullscreen switcher.
