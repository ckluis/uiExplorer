v9: "Task Triage" — Swipeable Card Stack

Tagline: "Swipe right to fix, left to defer."

Core Concept: A Tinder-style swipeable card stack for batch processing issues during downtime. Gamified with streak counters and deck depth indicators. Makes batch processing feel satisfying.

Exposure Method: Full-screen card stack experience, accessible from dedicated "Triage Mode" button.

Presentation Style: Stacked cards (visible deck behind current card). Swipe gestures indicated visually. Current card prominently displayed.

Target Persona: Mobile power user, batch-processing during downtime (commute, waiting). Wants to clear the deck efficiently.

Platform: Mobile-first (375x720). Desktop shows phone frame.

Key UI Elements:
- Card stack with visible depth (3-4 cards showing behind)
- Swipe gesture indicators (left Defer, right Fix, up Escalate)
- Deck depth counter ("5 of 23 remaining")
- Streak indicator ("7 in a row!")
- Undo toast after each action
- Priority badges on cards
- Quick action buttons below card (for non-swipe users)
- Progress bar showing deck progress
- "Deck cleared!" celebration state

Sample Cards in Stack:
1. "Update API endpoint URL" — Priority: High
2. "Review access permissions" — Priority: Medium
3. "Acknowledge SSL warning" — Priority: Low
4. "Reset failed webhook" — Priority: High
5. "Update billing contact" — Priority: Medium

Color/Mood: Gamified, energetic. Card aesthetic with shadows showing stack depth. Satisfying visual feedback on actions. Deep purple-to-blue gradient background.

Interaction Patterns:
- Radio buttons simulate current card in stack
- CSS transforms for card positioning (translateX, rotate)
- Checkbox states for swipe direction hints
- Toast animation for undo feedback
- Counter updates via :checked cascades

Device Preview: Include switcher, but default to mobile view.
