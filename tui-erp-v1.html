TUI ERP

v1

Purpose

Reimagines the platform-dashboard ERP as a terminal user interface — monospace type, box-drawing borders, ANSI colors, ASCII data viz, and keyboard shortcuts everywhere. A brutal rejection of modern UI polish in favor of density and muscle memory.

Prompt

Let's build a new experiment. Let's build an interface reminscent of a TUI, but with slightly more polish.

Specifically re-imaging the erp from the platform-dashboard series of experiments, but now inside a TUI experiment. This means many of the patterns we used need to be completely rethought.

This should be a no holds bar attack on modern design and eschew modern niceties. It needs to be brutally focused on muscle memory and keyboard commands (although they can be non-operational since we will not use js). Go hard.
